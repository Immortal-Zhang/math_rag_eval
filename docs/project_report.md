# 项目报告：数学资料检索问答与评测系统

## 1. 项目目标

本项目围绕数学课程资料构建一个小规模检索增强问答原型，目标不是训练大模型本身，而是把“文档整理 → 文本切块 → 检索 → 回答生成 → 评测 → 页面演示”这条链路完整跑通，并形成可复现、可展示、可分析的学生项目。

## 2. 数据来源与规模

当前版本使用 `data/raw/source_txt/` 下的数学资料文本，共 12 篇，覆盖以下主题：

- 压缩映射原理
- 黎曼积分与勒贝格积分
- 梯度下降的收敛性
- 牛顿法
- 最小二乘法
- 奇异值分解
- 拉格朗日乘子法
- 马尔可夫链
- 一致连续与连续
- 傅里叶级数
- 凸函数与强凸函数
- 常微分方程存在唯一性

评测问题保存在 `data/raw/eval_queries.jsonl` 中，共 24 条。每条样本包含：

- `query_id`
- `query`
- `relevant_doc_id`
- `reference_keywords`

在当前默认参数下，系统会先将原始资料整理为 `docs.jsonl`，再切块生成 `chunks.jsonl`，作为后续检索与回答模块的输入。

## 3. 方法流程

### 3.1 文档整理

使用 `prepare_corpus.py` 读取 `data/raw/source_txt/` 下的原始文本，清洗多余空白字符，并统一整理为 `data/raw/docs.jsonl`。

### 3.2 文本切块

使用 `chunk_docs.py` 对长文本进行切块，默认参数为：

- `chunk_size = 220`
- `overlap = 40`

采用重叠切块的原因是减少定义句、条件句或关键结论在边界处被切断后导致的信息损失。

### 3.3 检索

当前项目支持两种检索后端：

- `TF-IDF`：使用中文字符级 n-gram，适合术语型短查询
- `SentenceTransformer`：用于更语义化的向量检索（本版本以 `TF-IDF` 为主）

本次项目展示和评测主要基于 `TF-IDF` 后端完成。

### 3.4 回答生成

当前 baseline 为规则式回答生成。系统先检索 top-k 证据块，再优先抽取与问题关键词相关的句子，最终拼接为回答结果。这样做的目的是在没有额外大模型接口的条件下，先完整验证“检索 + 回答”的应用闭环。

项目中还预留了 `generate_answer_api.py`，后续可以接入外部大模型接口，在检索证据基础上生成更自然的回答。

### 3.5 评测

本项目采用两类评测指标：

#### 检索评测
- `Recall@1`
- `Recall@3`
- `Recall@5`

#### 回答评测
- 关键词覆盖率
- 人工打分模板

其中，Recall@K 用于判断正确文档是否出现在前 K 个检索结果中；关键词覆盖率用于衡量回答是否覆盖参考答案中的核心概念。

## 4. 当前结果

以当前示例语料、`TF-IDF` 后端和默认参数为例，项目跑通后的结果如下：

- `R@1 = 0.9583`
- `R@3 = 1.0000`
- `R@5 = 1.0000`
- `平均关键词覆盖率 = 0.8125`

对应结果文件位于：

- `artifacts/retrieval_eval_tfidf.csv`
- `artifacts/retrieval_summary_tfidf.json`
- `artifacts/answer_eval_tfidf.csv`
- `artifacts/answer_summary_tfidf.json`
- `artifacts/sample_retrieval.txt`
- `artifacts/sample_answer.txt`

这些结果说明：在当前小规模原型数据上，系统已经能够较稳定地完成资料检索与基础回答生成。

## 5. 项目价值

本项目的主要价值不在于模型规模，而在于完成了一条可解释、可评测、可演示的 RAG 应用闭环。通过这个项目，可以展示以下能力：

- 能将原始文本资料整理成统一可处理格式
- 能完成文本切块、索引构建和检索流程
- 能基于检索证据实现基础回答生成
- 能设计并实现离线评测指标
- 能使用页面工具完成结果演示

因此，这个项目更适合作为“大模型应用 / RAG / 评测 / Agent 基础能力”方向的学生项目，而不是作为底层预训练项目来展示。

## 6. 局限与 bad case 分析

虽然当前原型已经跑通，但仍存在一些局限：

1. 当前语料规模较小，结果更适合说明方法闭环可行，而不是证明大规模泛化能力。  
2. 当前回答生成以规则式 baseline 为主，表达能力和概括能力有限。  
3. 某些问题虽然可以检索到相关资料，但回答可能只覆盖局部定义，遗漏充分条件或补充说明。  
4. 当查询表达过于简略时，检索排序可能受到关键词重合度的影响。  
5. 切块边界仍可能导致个别关键句被拆分，影响证据完整性。  

## 7. 后续改进方向

后续可以从以下几个方向继续扩展：

- 扩大数学资料规模，增加更多主题与问法
- 增加 reranker，提高 top-k 排序质量
- 引入更强的向量检索后端，对比 TF-IDF 与语义检索效果
- 接入 API 版回答生成，对比规则式 baseline 与大模型增强版表现
- 为回答结果加入引用证据位置，增强可解释性
- 增加更完整的人工评分数据与 bad case 分类分析

## 8. 总结

本项目完成了一个小规模数学资料 RAG 原型，能够围绕固定资料完成资料整理、切块、检索、回答生成、离线评测和页面演示。当前版本仍以原型验证为主，但已经具备作为简历项目和面试项目展示的基本条件。
